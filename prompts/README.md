# Academic Paper Generation Pipeline — Quick Start Guide

## ファイル構成

```
prompts/
├── 00_MASTER_ORCHESTRATOR.md   ← パイプライン全体の設計・運用ガイド
├── 01_FIELD_CARTOGRAPHY.md     ← Stage 1: 学術分野の調査・ギャップ分析
├── 02_HYPOTHESIS_GENERATION.md ← Stage 2: ブレイクスルー仮説の生成
├── 03_CONTINUOUS_IMPROVEMENT.md← Stage 3: 3ラウンドの継続的改善ディスカッション
├── 04_FORMAL_PROOFS.md         ← Stage 4: 形式的証明の構築
├── 05_PAPER_DRAFT.md           ← Stage 5: 論文ドラフト全体の執筆
├── 06_PEER_REVIEW.md           ← Stage 6: 3名の独立ピアレビューシミュレーション
├── 07_REVISION.md              ← Stage 7: レビュー対応・改訂
├── 08_FINAL_OUTPUT.md          ← Stage 8: 最終HTML出力生成
├── MEGA_PROMPT_ALL_IN_ONE.md   ← 全ステージ統合版（1回のセッションで実行）
└── README.md                   ← このファイル
```

## 使い方

### 方法A: 一括実行（推奨：長いコンテキストウィンドウを持つモデル向け）

1. `MEGA_PROMPT_ALL_IN_ONE.md` を開く
2. 末尾の `[INSERT YOUR RESEARCH TOPIC...]` をあなたのトピックに置き換える
3. AI（Claude, GPT-4, Gemini等）に貼り付けて実行
4. 8ステージすべてが順次実行される

### 方法B: ステージ別実行（推奨：最高品質を求める場合）

1. `01_FIELD_CARTOGRAPHY.md` のプロンプトにトピックを入力して実行
2. Stage 1の出力を保存
3. `02_HYPOTHESIS_GENERATION.md` のプロンプトにStage 1の出力を貼り付けて実行
4. 以降、各ステージの出力を累積的に次のステージに渡す
5. **異なるAIモデル**を各ステージで使い分けることで多様性を確保

### 方法C: マルチモデル戦略（推奨：最も確実）

| ステージ | 推奨モデル | 理由 |
|---------|-----------|-----|
| 1 (分野調査) | Claude / GPT-4 | 広範な知識、繊細な調査 |
| 2 (仮説生成) | Claude / Gemini | 創造的推論、分野横断 |
| 3 (議論) | ローテーション | 多様な視点 |
| 4 (証明) | Claude / GPT-4 | 数学的厳密性 |
| 5 (執筆) | Claude | 長文の一貫性 |
| 6 (査読) | 3つの異なるモデル | 査読者の独立性 |
| 7 (改訂) | Stage 5と同じ | 一貫性 |
| 8 (最終出力) | Claude | HTML/LaTeX生成 |

## 品質基準

最終論文は以下を満たすこと：
- ✅ 真に新規な理論的貢献（既存の言い換えではない）
- ✅ 少なくとも1つの反証可能な数値的予測
- ✅ すべての定理に完全な証明
- ✅ 具体的な実験プロトコル
- ✅ 100点満点中90点以上の総合評価
- ✅ ブラウザで直接開ける自己完結型HTML出力

   - 期待: 情報損失最小化の最適アーキテクチャ設計原理
