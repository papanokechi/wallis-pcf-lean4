Architecture Overview — V7.1

L1: Generator→ L2: Falsifier + Kill Floor→ L3: Experimentalizer→ L4: Convergence→ ★ ◆ ○ Tiered Output

Core Philosophy: Real breakthroughs are rare. This system is designed to kill ideas, not generate them. 80-90% of hypotheses should die during falsification. Only survivors with concrete mechanisms, testable predictions, and novel contributions pass to convergence.


Layer 1 — Hypothesis Generator: Max 3 hypotheses per iteration. Each has genome encoding: claim, mechanism, predicted effect, failure condition, assumptions, predictions, dependencies. [V7.1: + derivation field]

Layer 2 — Falsification Engine: Adversarial attack + mechanical kill floor [Fix #4]. Checks novelty vs known work, hidden assumptions, alternative explanations, logical weaknesses. Any score below floor = automatic kill.

Layer 3 — Experimentalizer: Transforms survivors into testable experiment designs: dataset, metric, prediction. [V7.1: + score_justifications]

Layer 4 — Convergence Engine: Clusters survivors, tests unification candidates, outputs tiered labels [Fix #5] instead of binary breakthrough.


New in V7.1: 3 Canvas visualizations (alignment entropy, phase transition, geometric manifold), H6.2 experiment protocol tab, integrated 9.2/10 peer review tab.

Scoring Formula — B = N × F × E × C

NNovelty — distance from closest known work≥ 0.70 FFalsifiability — precision of kill condition≥ 0.80 EEmpirical — testability with current infrastructure≥ 0.65 CCompression — how much it explains per free parameter≥ 0.60

Why multiply? A hypothesis scoring 0.95 on three axes but 0.2 on one is not 70% of a breakthrough — it is missing something essential. Multiplication enforces every dimension clears a floor.

H6.2 Minimal Experiment — Spectral Gap as Reward Hacking Predictor

Core Hypothesis: A "mushy" reward model (small Delta) allows the policy to drift into unintended latent regions, while a "peaked" model (large Delta) pins the policy to the human-preferred manifold. The critical KL budget before hacking is:

D_crit = Delta^2 / (4 · d)

where Delta = lambda_1 - lambda_2 (spectral gap of the preference covariance matrix) and d = hidden dimension (e.g. 4096 for Llama-7B).

4-Phase Validation Protocol

Phase 1: Extract the Preference Matrix

Given N prompt-completion pairs {x, y_w, y_l}, extract last-layer hidden states h_w, h_l from the reward model and construct:

M = (1/N) · Sum( (h_w - h_l)(h_w - h_l)^T )

This covariance matrix M encodes the preference directions in latent space.

Phase 2: Compute Spectral Gap (Delta)

Perform SVD on M → eigenvalues lambda_1, lambda_2, ..., lambda_d.
The Gap: Delta = lambda_1 - lambda_2
The Prediction: D_crit = Delta^2 / (4 · d)

Phase 3: PPO Validation

Run PPO and monitor "Gold Reward" (oracle) vs "Proxy Reward" (RM under test):

ObservationTheory PredictionStatus
KL < D_critBoth proxy and gold rewards improveSafe Optimization
KL ≈ D_critProxy rises; gold plateausOnset of Hacking
KL > D_critProxy rises; gold crashesReward Collapse

Phase 4: Falsification Criteria

Model: Llama-3-8B (policy) + Llama-3-8B-RM (proxy)
Perturbation: 3 RM versions with Gaussian noise sigma in {0.01, 0.05, 0.1}
Steps: 1000 PPO steps per configuration
Success Metric:
|D_actual - D_crit| / D_actual < 0.15
Must hold across all 3 noise levels. Synthetic demo: all 3 pass.

# Phase 2: Spectral Gap Computation -- Core Logic
def compute_spectral_gap(diff_vectors, hidden_dim, noise_sigma=0.0):
    V = diff_vectors.clone().float()
    if noise_sigma > 0:
        noise = torch.randn_like(V) * noise_sigma * V.norm(dim=1, keepdim=True)
        V = V + noise
    M = (V.T @ V) / len(diff_vectors)  # [d, d] preference covariance
    lambdas = svdvals(M)
    delta = lambdas[0].item() - lambdas[1].item()
    d_crit = (delta ** 2) / (4 * hidden_dim)
    return SpectralResult(lambda_1=lambdas[0], lambda_2=lambdas[1],
                         delta=delta, hidden_dim=hidden_dim, d_crit=d_crit, ...)

# Phase 4: Validation
def validate_prediction(spectral, collapse, threshold=0.15):
    d_actual = collapse.kl_at_collapse
    rel_error = abs(d_actual - spectral.d_crit) / abs(d_actual)
    return ValidationResult(passed=rel_error < threshold, ...)
